# Wiki Creator - 开发计划汇总

本文档汇总了 Wiki Creator 项目的所有设计和实现计划。

---

## 项目概述

**Wiki Creator** 是一个将任意知识点转化为结构化、图文并茂 Wiki 的全栈应用。

输入任意知识点，自动搜索学术、技术、通用多源信息，结合大模型生成由浅入深的 4 层金字塔结构 Wiki。

---

## 核心特性

### 1. 多源权威信息聚合

**4 种数据源并行搜索：**

| 数据源 | 类型 | 说明 |
|--------|------|------|
| **Tavily** | 通用搜索 | 为 AI 应用优化的搜索引擎 |
| **Semantic Scholar** | 学术论文 | 学术论文数据库，高可信度 |
| **arXiv** | 预印本论文 | 最新研究前沿 |
| **GLM5 知识库** | LLM 知识 | 智谱 AI 大模型直接知识查询 |

**智能聚合策略：**
- 每种数据源贡献 top 20 结果（按可信度排序）
- 最大 80 条参考来源
- 中英文智能翻译（中文术语自动翻译为英文搜索学术 API）
- 失败/空数据源显示警告，不阻塞流程

### 2. 4 层金字塔知识结构

```
┌─────────────────────────────────────────────┐
│ Layer 1: 直觉层 (10秒理解)                  │
│ 一句话定义 + 生活类比 + 概念图               │
└─────────────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────┐
│ Layer 2: 原理层 (为什么能这样做)             │
│ 核心逻辑 + 关键推导 + 原理图                 │
└─────────────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────┐
│ Layer 3: 深入层 (完整理解，可折叠)           │
│ 完整推导 + 边界情况 + 概念关系对比           │
└─────────────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────┐
│ Layer 4: 应用层 (怎么用)                    │
│ 实际应用场景 + 代码示例 + 常见误区           │
└─────────────────────────────────────────────┘
```

### 3. 智能知识分类

系统自动识别知识点类型，针对性生成内容：

| 类型 | 特点 | 侧重点 |
|------|------|--------|
| 数学原理型 | 公式、定理 | 公式推导 + 直觉解释 |
| 物理原理型 | 物理定律 | 物理直觉 + 实验类比 |
| 架构/系统型 | 软件架构 | 组件关系 + 数据流 |
| 逻辑/算法型 | 算法、数据结构 | 步骤拆解 + 复杂度分析 |
| 概念/理论型 | 抽象概念 | 定义 + 历史 + 多角度对比 |
| 实践/工具型 | 工具、框架 | 使用场景 + 最佳实践 |

### 4. 图表可视化

**支持的图表类型：**

| 类型 | 使用场景 | 渲染方式 |
|------|----------|----------|
| Mermaid 流程图 | 执行步骤、系统架构、决策分支 | 客户端 mermaid.js |
| Mermaid 思维导图 | 概念层级、知识结构 | 客户端 mermaid.js |
| AI 配图 | 抽象概念、物理现象 | CogView API（智谱 AI） |

**智能图表选择：**
- LLM 根据内容自动决定图表类型
- 避免过度可视化，仅在关键理解点添加
- 严格遵守 Mermaid 语法规范

### 5. 现代加载动画

生成过程中显示：
- 脉冲动画图标
- 步骤文字淡入淡出
- 流动进度条
- 波浪呼吸圆点

## 技术架构

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Next.js 16 + React 19 | 全栈一体化 |
| UI 组件 | TailwindCSS + shadcn/ui | 现代简约风格 |
| 数据库 | SQLite + Drizzle ORM | 单文件，零运维 |
| LLM | GLM5 (智谱 AI) | 用户自带 API Key |
| 搜索 | Tavily + Semantic Scholar + arXiv | 多源聚合 |
| 图表 | Mermaid.js | 客户端渲染 |

### 核心工作流

```
用户输入知识点
      │
      ▼
┌─────────────────────────────────────────┐
│ Step 1: 多源并行搜索                      │
│ ├─ Tavily (通用)                         │
│ ├─ Semantic Scholar (学术)               │
│ ├─ arXiv (预印本)                        │
│ └─ GLM5 知识库 (LLM)                     │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│ Step 2: LLM 处理                         │
│ ├─ 智能筛选 + 可信度标注                  │
│ ├─ 知识盲区识别 + LLM 补充               │
│ └─ 知识点类型分类                         │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│ Step 3: 4 层内容生成                      │
│ ├─ Layer 1: 直觉层                       │
│ ├─ Layer 2: 原理层                       │
│ ├─ Layer 3: 深入层                       │
│ └─ Layer 4: 应用层                       │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│ Step 4: 图表决策 + 渲染                   │
│ ├─ 流程图 (Mermaid)                      │
│ ├─ 思维导图 (Mermaid)                    │
│ └─ AI 配图 (CogView)                     │
└─────────────────────────────────────────┘
      │
      ▼
    完成 Wiki
```

## 设计决策

### 为什么选择 4 层金字塔结构？

1. **渐进式理解**：从直觉到深度，满足不同水平读者
2. **费曼学习法**：能用简单语言解释，才算真正理解
3. **知识复用**：每层独立，可单独引用

### 为什么用 SQLite？

1. **零运维**：单文件数据库，无需独立服务
2. **部署简单**：随应用一起部署
3. **备份方便**：复制文件即可

### 为什么选择 GLM5？

1. **中文优化**：智谱 AI 对中文支持优秀
2. **成本可控**：用户自带 API Key
3. **功能完整**：支持文本生成、图像生成
